2 DESCRIPTION OF CLASSES

 

2.1 Artist Class

The artist class is the superclass of the Soloist Class and Band Class, which means an artist can either be a soloist or band. This class allow users to create an artist using the entered details of the artist. Fields include name and age.

 

Constructor: The constructor of the Artist class is to create a artist whose name is what the user entered.

 

Methods: This class includes three methods: getArtistName, setArtistAge and getArtistAge.

 

Click HERE to get the javadoc of this class.

 

 

2.2 Soloist Class

 

The soloist class is a subclass of the Artist class. A soloist can only have a name and age and cannot have members. Fields of the soloist class are all inherited from the Artist class.

 

Constructor: This constructor create the soloist whose name is what the user entered.

 

Methods: This class do not have any methods because all the methods are inherited from the superclass.

 

Click HERE to get the javadoc of this class.

 

 

2.3 Band class

 

The Band class is another subclass of the Artist class. Unlike Soloist class, a band need to add band member, so the band class need another ArrayList called bandMembers to store soloist objects.

 

Constructor: The constructor of the band class create the band using the name that the user entered and it will initialise the bandMembers ArrayList.

 

Methods: The Band class have a method called addBandMember to add soloists to the ArrayList. And it also have a method call getBandMember to return the whole ArrayList.

 

Click HERE to get the javadoc of this class.

 

2.4 Track class

 

The track class is defined to store all the information related to a music track. The information include track title, artists, guest artists, track rating, track length and recorded year etc.

 

Constructor: The constructor of the track class create a track using the name that the user entered. It also initialise the ArrayLists of trackArtist and guestArtist and set playCount to zero.

 

Methods: The track class contains methods to get and set the artists, guest artists, length，rating, location and size of the track. This class also have a method to return a list of all the individuals on the track.If the artist is a soloist, it would be directly returned. But is the artist is a band, the members of the band would be returned. The playTrack method is to simulate the play operation of the music tracks. Every time the playTrack method is called by the user, the playCount will add one automatically.

 

Click HERE to get the javadoc of this class.

 

2.5 Album class

 

The album class is defined to store all the information related to a music album. The information include album title, artists, guest artists, type and average rating. It also calculate the total running time and size of the album.

 

Constructor: The constructor of album class create a album and initialise the albumTrack ArrayList and albumArtist ArrayList. It also set the totalLength and averRating to zero.

 

Methods: The album class contains methods to get and set album type. it also has method to add tracks into the album. After adding the tracks we can get an ArrayList of artists inside the tracks just added. The album class also has a method to calculate total length and another method to transform seconds time format to “hh:mm:ss” format. The album class can also calculate total size and average rating of the album.

 

Click HERE to get the javadoc of this class.

 

 

2.6 ImportMP3 class

 

The ImportMP3 class is a class to help the MusicLibrary class to import real mp3 files from the given folder. I wrote this class by importing a jAudioTagger library into the bluej and using the methods in the JAudioTagger library. The jAudioTagger is a library to get  tag information of audio files and it fully supports mp3 files.

 

The fields in this class is quiet same as the fields in the track class including trackTitle, trackLength, trackRate, trackLocation and trackSize. It also has an array list to store tracks and an ArrayList to store albums. And I defined a tempTitle field to store the filename of the mp3 files.

 

Constructor: The constructor initialise the folder field.

 

Methods: 

(1) readTracks(final String suffix): This method is to read music file in the defined folder. The parameter suffix is the suffix of the files( in this case, all the suffix should be “.mp3”). Inside the readTracks method, I used one code introduced in one excise of BlueJ textbook to make sure all the files that being read have the matching suffix. And I use a for loop to add the tracks into the tracks ArrayList.

 

Figure 2 Use the musicLibrary object to import three real music tracks

 

(2) readAlbums(final String suffix): This method has the same idea as the read Tracks method. It can read album information of the mp3 files and return an ArrayList of albums with album title, album artist and album type. However, when I call this method in the MusicLibrary class, this method will add several same albums of the last music file into the album ArrayList and return it.

(4) getNewAlbum(File file): This method is built with the same idea as getNewTrack method, to help the readAlbums method to get the albums with information and tags.

 

(5) setMp3TrackSize(): This method is to get the size of the music track and use the size value to set field trackSize. There are also other methods to set the information of the music track like setMp3TrackTitle(), setMp3TrackDate(), setMp3TrackArtist(), setMp3AlbumTitle(), setMp3AlbumType().

2.8 MusicLibrary class

 

The MusicLibrary class is to create a music library to store albums tracks. It can make a random list of the added tracks. In this class, there is a variable ArrayList called allTracks to store all the tracks that are added into the library, include the tracks inside a added album.

 

Constructor: The constructor initialise the albums ArrayList and allTracks ArrayList.

 

Methods: 

(1) addAlbum(Album album): This method can add albums to the music library. The albums will be added into the albums ArrayList and the tracks inside the album will be added into the allTracks ArrayList.

 

(2) addTrack(Track track): Like addAlbum method, this method can add tracks into the allTracks ArrayList.

 

(3) importTracks(): This method will call the methods in the ImportMP3 class. First it defines two variable called folder and suffix. Folder is the folder to store the real tracks( in this case, it is “MP3” folder). The suffix the the “.mp3” suffix of real tracks. Then it will build a new ImportMP3 object using the folder variable and call the method readTracks(suffix) and readAlbums(suffix). The tracks and albums will be added into the ArrayList.

 

(4) creatPlayList(int minRating, int duration): This method is to create a random playlist that meet the use’s requirement. The minRating is the required minimum rating the user gave. Tracks in the playlist should have a rating no lower than the minimum rating. The duration is the required duration that the user gave. The total length of the tracks in the playlist should be no longer than the duration.

 

(5) the MusicLibrary class also has method called getTracks() to get a list of all the tracks inside the music library.
